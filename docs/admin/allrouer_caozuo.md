## 2,支付设置

### 2.1,易支付设置

1.使用管理员登录后左侧菜单->系统设置 ->支付设置。

![image-20260305141627798](./screenshot-20260310-180202.png)

2.配置易支付地址，固定为https://域名/epay-api（域名为易支付的地址，该易支付是和allrouter配套的系统），易支付部署请联系管理员，即可获得易支付地址，回调地址填写当前allrouter系统的域名即可。易支付商户id，易支付商户密钥请联系管理员。

3.充值价格设置，请按照实际美元汇率设置（x元/美金,指的就是x人民币=1美金），这个会影响实际的充值价格。例如：![image-20260305142424034](image-20260305142424034.png)

在钱包管理页面（左侧菜单找到即可就可以进入）![image-20260305142502650](./image-20260305142502650.png)

如果充值价格设置了7.32，在钱包管理页面的选择充值额度就会如图所示（计算公式为，拿如图的10$做例子就是10*充值价格/汇率设置）：![image-20260305142605442](./image-20260305142605442.png)

### 2.2,汇率设置

汇率设置，需要在系统设置->运营设置中，站点额度展示类型及汇率进行设置

![image-20260305143009143](image-20260305143009143.png)



![image-20260305143036291](image-20260305143036291.png)

选择CNY币种，修改人民币->美元的汇率7.3，改为7.32后点击保存通用设置后，再切换回美元展示再点击保存通用设置。

![image-20260305143623701](image-20260305143623701.png)

### 2.3,易支付管理端设置

访问 https://www.allrouter域名/epay-admin,输入账号密码登录

![screenshot-20260309-090623.png](screenshot-20260309-090623.png)

登录完成后在商户管理中输入商户名称后点击创建商户添加一个商户，系统自动生成pid已经这个商户的key。

![image-02](screenshot-20260309-091344.png)

将系统生成的pid和key填写到allrouter管理员端的系统设置->支付设置->支付设置中的易支付商户id，易支付商户密钥即可

![image-03](screenshot-20260309-092021.png)

## 3.添加模型总体逻辑

### 3.1,基准设置

**首先要对模型进行基准设置,也就是模型的输入和输出(基本上是官方的价格),如果没有此模型,那么就添加此模型**

1.设置基准

![image-20260309160627582](image-20260309160627582.png)

![ScreenShot_20260309-160831131.png](ScreenShot_20260309-160831131.png)

如果搜索不出来,那么就添加添加模型

![ScreenShot_2026-03-09_161518_995](ScreenShot_2026-03-09_161518_995.png)

可以搜索出来,那么就点击编辑,进行基准设置,最后一点要点击应用更改按钮

![ScreenShot_2026-03-09_161108_096](ScreenShot_2026-03-09_161108_096.png)

### 3.2,分组设置

新建分组,设置分组倍率,设置成功之后,该分组内的所有模型都会价格都是根据基准价格乘以设置倍率得来的

![ScreenShot_2026-03-09_162029_181.png](ScreenShot_2026-03-09_162029_181.png))

### 3.3,渠道设置

渠道设置专门用来配置模型，以claude为例。

1.管理员登录后 在左侧菜单列表找到渠道管理后，在渠道管理页面点击添加渠道开始添加模型。

![image-20260305151005717](image-20260305151005717.png)



2.填写渠道名称，这个渠道的claude模型的密钥，请求claude模型的api地址即可（注意claude api地址不需要填写/v1/messages,如果是gpt不需要填写/v1/chat/completions等末尾）

![image-20260305151329589](image-20260305151329589.png)

![image-20260305153505749](image-20260305153505749.png)

注意要在高级设置里面把自己新建的分组选择上,点击提交即可



![ScreenShot_2026-03-09_164037_630](ScreenShot_2026-03-09_164037_630.png)



3.模型重定向设置（非必须，以豆包为例）

![image-20260305154605508](image-20260305154605508.png)

如图所示，如果不设置模型重定向那么实际请求的模型就是Doubao-1.8,在模型广场展示的也是Doubao-1.8。如果设置了模型重定向，虽然模型广场展示的是Doubao-1.8，实际请求的是doubao-seed-1-8-251228。（这个场景用于，有的模型厂商要求模型请求是ENDPOINT_ID，例如ep-hxjxxxxxx，显示在模型广场用户会一脸懵逼的）

![image-20260305155256925](image-20260305155256925.png)

#### 3.3.1 需要将web服务伪装成claude code（可选）

1.管理员登录到控制台->渠道管理

![](screenshot-20260311-144654.png)

2.进入到渠道管理中点击添加渠道后弹出框中基本信息中类型选择 Anthropic Claude，填写api密钥，api地址(只需要填写https:域名无需填写/v1/messages后端会自己处理)选择相应的模型或填写自定义模型。

![](screenshot-20260311-144958.png)

![](screenshot-20260311-144917.png)

3.在高级设置中找到请求头覆盖，填写：

```json
{
  "x-api-key": "{api_key}",
  "User-Agent": "claude-cli/2.1.51 (external, cli)",
  "x-app": "cli"
}
```

![](20260311-145358.png)

4.找到渠道额外设置，Claude 强制 beta=true打开，即可将web服务伪装成claude code

![](screenshot-20260311-145543.png)

## 4,运营设置

### 4.1,新用户注册额度赠送设置

管理员登录后在左侧菜单栏->系统设置->运营设置中进行设置

![image-20260305155256925](screenshot-20260305-164014.png)

找到额度设置进行填写，保存额度设置即可（10000000Token(1千万)=20$）

![image-20260305155256925](screenshot-20260305-164137.png)

### 4.2,签到设置

管理员登录后在左侧菜单栏->系统设置->运营设置中进行设置

在运营设置中找到签到设置，设置相应的额度即可。（10000额度相当于0.01$,所以默认的额度区间是0.01$-0.1$）

![image-20260306133407218](image-20260306133407218.png)
