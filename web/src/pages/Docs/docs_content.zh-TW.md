AllRouter.AI 是一個統一的大模型閘道，旨在為您提供便捷、高效且低成本的 AI 模型接入服務。本手冊將從一般使用者的視角，帶您熟悉平台的核心功能與使用流程。

# 平台概覽

AllRouter.AI 整合了 OpenAI、Claude、Llama 等 50+ 知名大模型，透過統一 API 介面，讓您無需在多個平台之間切換即可調用完整 AI 能力。

核心優勢：

無縫替換：100% 相容 OpenAI SDK，只需修改 `base_url` 即可接入。

智慧路由：根據規則自動選擇成本更低或回應更快的模型。

高可用性：支援故障自動切換，確保服務不中斷。

即時觀測：控制台提供完整的延遲、Token 用量與成本統計。

# 快速入門

## 註冊登入

### 註冊

造訪 https://allrouter.shengjian.net/，點擊上方「登入」->「註冊」，在彈出頁面依序輸入使用者名稱與密碼，最後點擊「註冊」即可完成帳號建立。

![](/docs_images/img_2.png)

![](/docs_images/img_3.png)

### 登入

造訪 https://allrouter.shengjian.net/，點擊上方「登入」，依序輸入使用者名稱與密碼，點擊「繼續」即可成功登入 AllRouter.AI。

![](/docs_images/img_4.png)

## 操練場

功能描述：無需撰寫程式碼，直接在網頁端與各類 AI 模型互動測試。

1. 第 1 步：在左側選單點擊【操練場】。

2. 第 2 步：在左側【模型設定】面板中的【模型】下拉選單選擇目標模型（例如 `gpt4o`）。

3. 第 3 步：（可選）調整【Temperature】滑桿，控制輸出的隨機性。

4. 第 4 步：在右側對話區底部輸入框輸入您的問題。

5. 第 5 步：點擊輸入框右側【發送】按鈕。

前置條件：帳戶餘額大於 0。

預期結果：AI 即時回傳內容，並在下方顯示消耗詳情。

![](/docs_images/img_5.png)

## 控制台

### 數據看板

登入控制台後，首先會看到數據看板，這裡能直觀展示您的帳戶狀態：

當前餘額：即時查看帳戶剩餘額度。

使用統計：包含請求次數、消耗額度與 Token 消耗。

資源消耗：包含累計消耗額度與累計消耗 Tokens。

數據指標：包含平均 TPM、平均 RPM。

模型數據分析：以圖表展示不同模型的消耗分佈與調用趨勢。

![](/docs_images/img_6.png)

### 令牌管理

功能描述：管理用於 API 呼叫的密鑰（Token），支援額度限制與到期設定。

若要開始調用 API，您需要先建立令牌：

1. 第 1 步：點擊左側選單【令牌管理】。

2. 第 2 步：點擊【新增令牌】按鈕。

3. 第 3 步：設定令牌名稱、額度限制、到期時間與存取限制。

4. 第 4 步：建立成功後，複製生成的令牌（Key），即可在您的應用中使用。

![](/docs_images/img_7.png)

### 使用日誌

功能描述：記錄所有透過 AllRouter.AI 閘道發起的 API 呼叫詳情，方便審計與成本分析。

1. 第 1 步：在左側選單點擊【使用日誌】，頁面會顯示 API 呼叫記錄清單。

2. 第 2 步：設定查詢時間範圍。點擊日期選擇器，在日曆中選擇【開始時間】與【結束時間】。

3. 第 3 步：輸入篩選條件。在【令牌名稱】、【模型名稱】、【分組】或【Request ID】輸入關鍵字。

4. 第 4 步：執行查詢。點擊【查詢】後，清單會刷新並顯示符合條件的記錄。

5. 第 5 步：查看日誌詳情。可在清單中查看【時間】、【令牌】、【模型】、【花費】等資訊，點擊【詳情】查看完整請求與回應。

![](/docs_images/img_8.png)

### 繪圖日誌

功能描述：記錄 Midjourney 等繪圖模型任務執行情況，支援查看狀態、進度與生成圖片。

1. 第 1 步：在左側選單點擊【繪圖日誌】，頁面顯示「Midjourney 任務記錄」。

2. 第 2 步：搜尋指定繪圖任務。輸入【任務 ID】並選擇時間範圍後，點擊【查詢】。

3. 第 3 步：查看繪圖結果。點擊【結果圖片】欄位中的縮圖。

4. 第 4 步：檢查任務狀態。若任務失敗，可在【失敗原因】欄位查看錯誤資訊。

![](/docs_images/img_9.png)

### 任務日誌

功能描述：記錄系統中所有非同步任務（如批次處理、長時任務）的生命週期。

1. 第 1 步：在左側選單點擊【任務日誌】，進入任務記錄頁面。

2. 第 2 步：篩選任務記錄。輸入【任務 ID】或設定時間範圍，點擊【查詢】。

3. 第 3 步：分析任務耗時。對比【提交時間】與【結束時間】，了解處理效率。

4. 第 4 步：自訂欄位顯示。點擊右上角【欄位設定】按鈕，勾選需要顯示的欄位。

![](/docs_images/img_10.png)

# 模型廣場

功能描述：「模型廣場」是 AllRouter.AI 的核心資源庫，不僅展示平台支援的全部 AI 模型，還提供透明、即時的計費查詢。使用者可依官方倍率或充值後的實際成本，精準評估每個模型的調用代價。

1. 第 1 步：在頁面頂部導覽列點擊【模型廣場】。進入模型清單後，預設以卡片形式展示模型 ID 與 1M Tokens 價格。

2. 第 2 步：掌握兩種計費模式。

* 按量計費（Tokens）：針對文字對話模型，依輸入、補全與快取讀取的 Tokens 計費。
* 按次計費（Requests）：針對繪圖、任務等特定模型，每發起一次請求扣除固定費用。

3. 第 3 步：靈活切換價格顯示方式。

* 充值價格顯示：開啟後，系統依您的充值比例自動換算並顯示實際扣費金額。
* 倍率顯示：開啟後，模型卡片顯示相對官方價格的計費倍率。

4. 第 4 步：利用側欄精準定位模型。在左側【供應商】區域點擊特定圖示，或在【計費類型】選擇【按量計費】。

5. 第 5 步：搜尋並複製模型 ID。在頂部搜尋框輸入關鍵字，找到目標後點擊卡片右上角【複製】圖示。

6. 第 6 步：批次操作與視圖切換。

* 表格視圖：點擊右上角【表格視圖】，可用更緊湊清單比較多個模型價格。
* 預期結果：顯著提升多模型場景下的配置效率。

# 使用示例

以下以在 Claude Code 中使用 AllRouter 為例。

## 步驟一：安裝 Claude Code

前置條件：

* 您需要安裝 [Node.js 18 或更新版本](https://nodejs.org/en/download/)。
* MacOS 使用者建議使用 [nvm](https://github.com/nvm-sh/nvm) 或 [Homebrew](https://formulae.brew.sh/formula/node) 安裝 Node.js。不建議直接用安裝包，後續可能遇到權限問題。
* Windows 使用者還需安裝 [Git for Windows](https://git-scm.com/download/win)。

進入命令列介面，安裝 Claude Code：

npm install -g @anthropic-ai/claude-code

執行以下命令檢查安裝結果，若顯示版本號代表安裝成功：

claude --version

## 步驟二：設定 AllRouter

1. 第 1 步：註冊帳號

造訪 AllRouter 平台，點擊右上角「註冊/登入」，依提示完成註冊與登入流程。

2. 第 2 步：取得 API Key

登入後，進入個人中心，點擊 API Keys，建立新的 API Key。

要開始呼叫 API，您需要先在 AllRouter 平台建立令牌：

* （1）點擊左側選單【令牌管理】。
* （2）點擊【新增令牌】按鈕。
* （3）設定令牌名稱、額度限制、到期時間與存取限制。
* （4）建立成功後，複製生成令牌（Key）即可在應用中使用。

![](/docs_images/img_21.png)

複製 API Keys 資訊。

![](/docs_images/img_22.png)

3. 第 3 步：設定環境變數

在 MacOS、Linux 或 Windows 設定環境變數（以下以手動設定為例）。

支援 MacOS、Linux、Windows，但不同系統配置檔案路徑不同。請確保修改後 JSON 格式正確。

```bash
# 編輯或新增 `settings.json`
# MacOS & Linux：`~/.claude/settings.json`
# Windows：`使用者目錄/.claude/settings.json`
# 新增或修改 env 欄位
# 將 `your_allrouter_api_key` 替換為您上一步取得的 API Key
```

```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "your_allrouter_api_key",
    "ANTHROPIC_BASE_URL": "https://allrouter.shengjian.net/v1",
    "API_TIMEOUT_MS": "3000000",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": 1
  }
}
```

```bash
# 再編輯或新增 `.claude.json`
# MacOS & Linux：`~/.claude.json`
# Windows：`使用者目錄/.claude.json`
# 新增 `hasCompletedOnboarding`
```

```json
{
  "hasCompletedOnboarding": true
}
```

設定效果：

```env
ANTHROPIC_AUTH_TOKEN=令牌 API Keys 金鑰
ANTHROPIC_BASE_URL=https://allrouter.shengjian.net/v1
ANTHROPIC_MODEL=令牌名稱
```

![](/docs_images/img_23.png)

## 步驟三：開始使用 Claude Code

設定完成後，進入您的程式碼工作目錄，在終端執行 `claude` 命令即可開始使用 Claude Code。

若遇到「Do you want to use this API key」，請選擇 Yes。
