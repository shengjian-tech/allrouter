AllRouter.AI は、複数の大規模モデルを統合した AI ゲートウェイです。低コストかつ高効率に AI モデルへ接続できるよう設計されています。本ガイドは一般ユーザー向けに、主要機能と利用フローをわかりやすく説明します。

# プラットフォーム概要

AllRouter.AI は OpenAI、Claude、Llama など 50 以上の主要モデルを統合し、統一 API で提供します。複数プラットフォームを切り替えることなく、幅広い AI 機能を利用できます。

主な特長：

シームレス置換：OpenAI SDK と 100% 互換。`base_url` の変更だけで接続可能。

スマートルーティング：ルールに基づいて低コストまたは高速応答モデルを自動選択。

高可用性：障害時の自動フェイルオーバーに対応し、サービス継続性を確保。

リアルタイム可観測性：コンソールで遅延、Token 使用量、コストを詳細に可視化。

### 登録

https://allrouter.shengjian.net/ にアクセスし、上部の **ログイン** -> **登録** をクリックします。表示された画面でユーザー名とパスワードを入力し、**登録** を押すとアカウント作成が完了します。

https://allrouter.shengjian.net/ にアクセスし、**ログイン** をクリックしてユーザー名・パスワードを入力し、**続行** を押すと AllRouter.AI にログインできます。

## プレイグラウンド

機能説明：コードを書かずに、Web 上で各種 AI モデルとの対話テストを行えます。

1. 手順 1：左側メニューで【プレイグラウンド】をクリック。

2. 手順 2：左側【モデル設定】パネルの【モデル】ドロップダウンから対象モデル（例：`gpt4o`）を選択。

3. 手順 3：（任意）【Temperature】スライダーで出力のランダム性を調整。

4. 手順 4：右側チャットエリア下部の入力欄に質問を入力。

5. 手順 5：入力欄右側の【送信】ボタンをクリック。

前提条件：アカウント残高が 0 より大きいこと。

期待結果：AI が即時に応答し、下部に消費詳細が表示されます。

![](/docs_images/img_5.png)

## コンソール

### ダッシュボード

ログインしてコンソールに入ると、最初にダッシュボードが表示されます。アカウント状況を直感的に確認できます。

現在残高：残高をリアルタイム表示。

利用統計：リクエスト回数、消費額、Token 消費量。

リソース消費：累計消費額、累計 Token 消費量。

データ指標：平均 TPM、平均 RPM。

モデル分析：モデル別の消費分布と利用トレンドをグラフ表示。

![](/docs_images/img_6.png)

### トークン管理

機能説明：API 呼び出しに使うキー（Token）を管理。上限設定や有効期限設定に対応。

API 呼び出し開始にはトークン作成が必要です：

1. 手順 1：左側メニューで【トークン管理】をクリック。

2. 手順 2：【トークン追加】をクリック。

3. 手順 3：トークン名、利用上限、有効期限、アクセス制限を設定。

4. 手順 4：作成後、生成されたキーをコピーしてアプリで利用。

![](/docs_images/img_7.png)

### 利用ログ

機能説明：AllRouter.AI ゲートウェイ経由のすべての API 呼び出しを記録し、監査やコスト分析に活用できます。

1. 手順 1：左側メニューで【利用ログ】をクリックし、呼び出し一覧を表示。

2. 手順 2：日時フィルターで【開始時刻】と【終了時刻】を設定。

3. 手順 3：【トークン名】【モデル名】【グループ】【Request ID】で絞り込み条件を入力。

4. 手順 4：【検索】をクリックして結果を更新。

5. 手順 5：一覧で【時刻】【トークン】【モデル】【コスト】などを確認し、【詳細】で完全なリクエスト/レスポンスを表示。

# モデルマーケット

機能説明：**モデルマーケット** は AllRouter.AI の中核リソースです。対応モデルを一覧表示し、透明でリアルタイムな料金確認を提供します。公式倍率やチャージ後実コストに基づき、各モデルの呼び出しコストを正確に評価できます。

1. 手順 1：上部ナビゲーションで【モデルマーケット】をクリック。初期表示はカードビューで、モデル ID と 1M Tokens 単価を表示。

2. 手順 2：2 つの課金モードを理解。

* Token 課金：テキスト対話モデルに対して、入力・補完・キャッシュ読み取り Token 数で課金。
* リクエスト課金：画像生成やタスク系モデルなどで、1 リクエストごとに固定費用を課金。

3. 手順 3：価格表示モードを切り替え。

* チャージ後価格表示：チャージ比率に基づいて実際の差し引き額を表示。
* 倍率表示：公式価格に対する課金倍率を表示。

4. 手順 4：サイドバーでモデルを絞り込み。左側【プロバイダー】または【課金タイプ】の【Token 課金】を選択。

5. 手順 5：検索・ID コピー。上部検索欄でキーワード入力し、対象カード右上の【コピー】をクリック。

6. 手順 6：一括確認と表示切替。

* テーブルビュー：右上の【テーブルビュー】で複数モデルを一覧比較。
* 期待結果：マルチモデル環境での設定効率が大きく向上します。

# 利用例

ここでは Claude Code で AllRouter を利用する方法を例示します。

## ステップ 1：Claude Code のインストール

前提条件：

* [Node.js 18 以上](https://nodejs.org/en/download/)をインストールしてください。
* macOS は [nvm](https://github.com/nvm-sh/nvm) または [Homebrew](https://formulae.brew.sh/formula/node) での導入を推奨します。
* Windows は [Git for Windows](https://git-scm.com/download/win) も必要です。

ターミナルで実行：

npm install -g @anthropic-ai/claude-code

インストール確認：

claude --version

## ステップ 2：AllRouter の設定

1. 手順 1：アカウント登録

AllRouter にアクセスし、右上の【登録/ログイン】から案内に従って登録・ログインします。

2. 手順 2：API Key の取得

ログイン後、個人センターで **API Keys** を開き、新しい API Key を作成します。

API 呼び出しには、事前に AllRouter でトークンを作成してください：

* （1）左メニューで【トークン管理】を開く。
* （2）【トークン追加】をクリック。
* （3）トークン名、上限、有効期限、アクセス制限を設定。
* （4）生成されたキーをコピーして利用。

![](/docs_images/img_21.png)

API Keys 情報をコピーします。

![](/docs_images/img_22.png)

3. 手順 3：環境変数の設定

macOS / Linux / Windows で環境変数を設定します（以下は手動設定例）。

システムごとに設定ファイルのパスが異なるため、JSON 形式が正しいことを確認してください。

```bash
# `settings.json` を編集または新規作成
# macOS & Linux: `~/.claude/settings.json`
# Windows: `<ユーザーディレクトリ>/.claude/settings.json`
# `env` フィールドを追加または更新
# `your_allrouter_api_key` を取得済み API Key に置換
```

```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "your_allrouter_api_key",
    "ANTHROPIC_BASE_URL": "https://allrouter.shengjian.net/v1",
    "API_TIMEOUT_MS": "3000000",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": 1
  }
}
```

```bash
# 次に `.claude.json` を編集または新規作成
# macOS & Linux: `~/.claude.json`
# Windows: `<ユーザーディレクトリ>/.claude.json`
# `hasCompletedOnboarding` を追加
```

```json
{
  "hasCompletedOnboarding": true
}
```

設定結果：

```env
ANTHROPIC_AUTH_TOKEN=トークン API Key
ANTHROPIC_BASE_URL=https://allrouter.shengjian.net/v1
ANTHROPIC_MODEL=トークン名
```

![](/docs_images/img_23.png)

## ステップ 3：Claude Code を使い始める

設定後、作業ディレクトリで `claude` コマンドを実行すると利用開始できます。

「Do you want to use this API key」と表示された場合は **Yes** を選択してください。

![](/docs_images/img_24.png)
